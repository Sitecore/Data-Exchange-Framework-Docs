Mappings Applied Action¶

Description A mappings applied action is a component that represents logic that is executed after a Value Mapping Set is applied successfully.
Template location /sitecore/templates/Data Exchange/Framework/Data Access/Mappings Applied Actions/Base Templates/Base Mappings Applied Action

In this topic:

  • Configuration
  • When to Extend

Configuration¶

The base mappings applied action should not be used on its own. It is a base template for specific types of mappings applied actions.

All mappings applied action inherit the following fields:

Field Description
Rules The Mappings Applied Action Rule that must pass in order for the mappings applied action to run. If no mappings applied action rules are specified, mappings applied action will always run.

See also

For information on the mappings applied actions included with the supported providers, see the section Mappings Applied Actions in the Component Reference.

When to Extend¶

Data Exchange Framework comes with a set of standard mappings applied actions. In addition, each provider may include a set of mappings applied actions designed specifically for the system the provider integrates.

Custom mappings applied actions can be implemented in cases where special logic must be run after a value mapping set is applied successfully.

See also

For information on how to implement a custom mappings applied action, see the section Custom Mappings Applied Action in the Cookbooks.
